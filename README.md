# 🌏 项目代号：Source of Truth (真理之源)

> **—— 拒绝二手噪音，一键直达技术源头。**

### 1. 💡 产品观 (Product Philosophy)

在这个 AI 生成代码泛滥、技术营销号满天飞的时代，开发者最稀缺的不是“答案”，而是**“准确性”**。

* **回归源头 (Back to Source)**：我们要解决的核心问题是“去哪里验证 AI 给我的代码对不对？”。只有官方文档才是唯一的真理标准（Single Source of Truth）。
* **技术中立 (Tech Neutrality)**：不参与无休止的“语言战争”。Java 也好，Python 也罢，我们不评判优劣，只提供通往它们核心知识库的最短路径。
* **极简主义 (Minimalism)**：不设广告，没有冗余的教程推荐。只做一件事：**让开发者在 3 秒内找到某个技术栈的最新 API 手册或官方指南。**

### 2. 📖 创作背景 (Origin Story)

作为一名拥有多年经验的开发者，我在从客户端向后端与 AI 转型时发现了一个有趣的现象：

起初，我想为 Java 编写一份中文文档，但我意识到：

1. **官方文档其实已经存在**，只是常被埋没在复杂的官网层级中。
2. **偏见阻碍了学习**，很多人因为听说“Java 繁琐”或“Go 简单”而陷入争论，却很少沉下心看一眼官方的 Design Principles。
3. **AI 的幻觉**，Chatbot 能生成代码，但它无法对代码的长期可维护性负责。

于是我放弃了“造轮子（写文档）”，决定做一个“指路牌”。**我不生产知识，我只是高效知识索引的搬运工。** 我希望为像我一样在技术转型期迷茫的开发者，提供一个干净、纯粹的导航站。

### 3. 🛠️ 技术方案 (Technical Stack)

为了贯彻“轻量、快速、易维护”的原则，本项目采用 **Jamstack** 架构：

* **核心框架**：`VuePress 2` + `VuePress Theme Hope`
* *理由*：拥有强大的文档与博客整合能力，内置美观的卡片布局和图标支持，开箱即用。


* **部署托管**：`GitHub Pages` + `GitHub Actions`
* *理由*：自动化构建，零成本托管，利用 Git 进行版本管理，不仅是网站，更是一个开源项目。


* **数据管理**：`YAML/JSON` 驱动
* *理由*：将导航链接与页面逻辑分离。未来任何人想提交新的语言文档，只需提一个 PR 修改数据文件即可，无需懂前端代码。


* **特色功能设计**：
* **Deep Links（深度链接）**：每个卡片不仅链接官网，更直接链接到 `Reference/API` 和 `Getting Started` 页面。
* **Version Tag（版本标记）**：明确标注文档对应的版本（如 Java 21 LTS, Python 3.12），拒绝过时信息。
