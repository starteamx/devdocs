# 🌍 产品白皮书：Source of Truth (真理之源)

> **Slogan:** 拒绝二手噪音，一键直达技术源头。
> **Internal Code:** DevOrigins

## 1. 🎯 产品愿景 (Product Vision)

在 AI 生成代码泛滥、SEO 垃圾农场充斥搜索引擎的时代，开发者最稀缺的资源不再是“答案”，而是**“准确性”**与**“判断力”**。

**Source of Truth** 不生产内容，不做简单的文档搬运。我们致力于构建一张**“开发者的高效导航地图”**，帮助从入门到资深的工程师完成从 **“怎么写”** 到 **“怎么选”** 的认知闭环：

1. **避开噪音**：跳过营销首页和第三方教程，直达官方 API 手册。
2. **建立体系**：通过结构化的知识图谱，理解技术的全貌。
3. **完成闭环**：不仅提供文档，更提供“业务场景”与“生态工具”的决策辅助。

## 2. 💎 核心价值主张 (The Trinity Value)

我们要打造的是一个 **“技术选型与查阅的铁三角”**：

* **📘 真理 (The Truth - Docs)**
* *解决问题：* “代码怎么写？”
* *方案：* **Deep Link (深链)**。拒绝浅层导航，所有链接直达核心 API 列表或规范文档 (Specification)，而非官网首页。


* **🎯 场景 (The Scope - Scenarios)**
* *解决问题：* “什么时候该用它？”
* *方案：* **Sweet Spot (核心打击区)**。明确每种语言 ROI 最高的 2-3 个核心业务场景（如 Python 之于 AI，Go 之于云原生）。


* **🧰 兵器 (The Gear - Ecosystem)**
* *解决问题：* “搭配什么工具用？”
* *方案：* **Must-Have List (军火库)**。只推荐经过时间验证的、事实标准级的框架与工具（如 Java 必配 Spring Boot）。

### 3.2 详情页策略：可视化与结构化

我们不做文档镜像（Mirror），我们做**“博物馆导览图”**。

* **知识图谱 (Knowledge Graph)**：基于官方目录绘制 Mermaid 导图，点击节点直达官方章节。
* **生态矩阵 (The Matrix)**：在每个语言页展示“核心场景”与“代表性生态”列表，辅助技术选型。

## 4. 📱 交互与 UI 设计 (UX Design)

### 4.1 首页：TIOBE 动态仪表盘

* **视觉重点**：卡片式网格布局，每个卡片右上角悬挂 **Rank Badge (排名角标)**（金/银/铜色）。
* **核心交互 (One Card, Three Paths)**：
每个卡片提供三个固定维度的直达按钮：
1. 🔴 **Manual**：直达 API Reference（高亮主按钮）。
2. 🔵 **Tutorial**：直达官方 Getting Started。
3. ⚪ **Web**：直达官方首页。


* **底部功能**：**决策对照表 (Decision Matrix)**，一句话解答“做微服务选谁”、“写脚本选谁”。

### 4.2 详情页/翻转卡片

展示该语言的 **“灵魂代码 (Soul Code)”** 片段（展示核心优势语法）以及 **“必装军火库”**。

## 5. 🛠 技术架构 (Technical Stack)

采用 **Jamstack** 架构，确保极致的轻量化、高性能与低维护成本。

* **核心框架**：`VuePress 2` + `VuePress Theme Hope`
* *理由*：开箱即用的文档布局，Markdown 扩展支持 Mermaid 图表。


* **数据驱动**：`TypeScript Interface`
* 所有导航数据（排名、链接、场景、生态）存储于 `src/data/languages.ts`。
* UI 组件通过遍历 JSON 数据生成，**内容与代码分离**，便于非技术人员维护内容。


* **部署**：GitHub Pages + GitHub Actions (CI/CD)。

## 6. 📅 路线图 (Roadmap)

* **Phase 1 (MVP):** 上线 Top 12 语言导航，实现“一卡三链”与排名角标。
* **Phase 2 (Content):** 完善 Java, Python, Go 的“知识图谱”与“生态矩阵”数据。
* **Phase 3 (Expansion):** 引入“决策对照表”页面，根据反馈增加 Docker/K8s 等工具栈。
